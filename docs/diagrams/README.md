## üé® Diagram Conventions

### Color Coding

- **Blue** (#4A90E2): System components, entry points
- **Green** (#51CF66): Success states, optimizations, benefits
- **Red** (#FF6B6B): Errors, traditional approaches, warnings
- **Yellow** (#FFD43B): Warnings, intermediate states
- **Purple** (#BD10E0): MCP-related, execution environments
- **Orange** (#F5A623): Processing, transformations

### Symbols

- **Solid arrows** (‚Üí): Direct flow, required steps
- **Dotted arrows** (-.->): Optional flow, references
- **Subgraphs**: Logical groupings, components
- **Notes**: Important clarifications, context
